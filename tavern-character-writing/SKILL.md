# 酒馆角色卡人设编写

## 核心原则

**写「成因链」，不写「标签清单」。** 给 AI 因果逻辑而非行为指令——理解「为什么」的模型能自行判断任何情境，死记「是什么」的模型只会机械复读。

**「角色喜欢玩家」只是一句话。「角色为什么喜欢玩家，她会怎样表达喜欢」才是一个角色的灵魂。**

标签式写法给 AI 一堆标签，但不告诉标签之间的关系和来源。AI 看到标签会在随机场景机械套用——不理解背后的心理，要么用错场景，要么复读到令人厌烦。

成因链写法给 AI 因果逻辑。AI 理解了「为什么」，就能在任何情境下自己判断角色会怎么做，而不是死记硬背几句台词。

---

## 强制工作方式：定位 → 成因链 → 性格树 → 行为语料 → 非性格内容

所有涉及角色卡/人设编写的任务，必须按五步法顺序执行。不允许跳步。

完整每一步的详细说明、判定标准和示例，见 [`references/method.md`](references/method.md)。

### 五步速览

| 步骤 | 做什么 | 核心产出 |
|---|---|---|
| 一、定位核心性格 | 找出角色最鲜明、最概括的性格内核 | 1-2 个核心性格，能解释大部分行为 |
| 二、写成因链 | 底色→塑形→表现，用「因为…所以…」串联 | 三层因果链条，每层之间有明确因果 |
| 三、延伸性格树 | 从核心性格延伸出衍生性格和反向性格 | 每条衍生标注成因，挂在核心上 |
| 四、补充行为语料 | 为具体行为和语料补充成因 | 行为挂在对应性格下面，说明为什么 |
| 五、非性格内容 | 角色名、社交关系、外貌穿着等基本信息 | 有性格影响的信息也写成因果关系 |

接手任务时，**必须先读取 `references/method.md`** 获取完整方法论。

## 快速参考

| 原则 | 做法 |
|---|---|
| **成因链** | 每个性格/特征都写出「因为X所以Y」，不堆标签 |
| **五步法** | 核心性格 → 成因链（底色→塑形→表现）→ 性格树 → 行为语料 → 非性格内容 |
| **性格树** | 核心→衍生→再衍生，每一条都标注成因 |
| **抽象优先** | 先写「为什么」，再补充「做什么」 |
| **解释 > 标签** | 每个概念都要阐释，不说「是什么」而说「为什么是这样」 |
| **非必要不补充** | 模型能自行演绎的内容不写进卡里 |
| **保留人味** | 用户的原始表达是最珍贵的素材，不可替换 |
| **一事一条** | 每个世界书条目只说一件事 |
| **正面指导** | 写「角色应该XXX」，不写「禁止角色XXX」 |

## 常见错误速览

| 错误 | 正确做法 |
|---|---|
| 只贴性格标签不给解释 | 每个标签都要阐释「这个性格体现在哪里」，写出成因 |
| 只写「是什么」不写「为什么」 | 从成因链出发，解释每个特征怎么来的 |
| 衍生性格独立于核心性格 | 每条衍生都要标注从核心的哪个部分延伸出来 |
| 先写具体例子再写抽象原因 | 先写抽象原因，再用具体例子补充 |
| 把用户想法交给 AI 重新生成 | 帮助整合意图，但必须用用户的话重新写 |
| 行为表现和语料单独放一个条目 | 放在对应性格下面，标注成因 |
| 写具体台词或固定动作让 AI 背诵 | 写行为逻辑，让 AI 自行演绎 |
| 一个条目塞进所有信息 | 一事一条，避免误触和注意力分散 |
| 写「禁止角色 XXXX」 | 写「角色应该 XXXX」 |
| 底色/塑形/表现之间没有因果 | 每层之间必须用「因为……所以……」串联 |

详细判定与修复方案见 [`references/checklist.md`](references/checklist.md)。

# 写作原则

**抽象优于具体**：先写行为背后的原因，再补充具体行为。模型能自行演绎的内容不需要写进卡里。

- ❌ 「她会在 user 帮助她时反而恶语相加，很毒舌」
- ✅ 「她心里喜欢 user 但是不敢表达，不能坦率地面对自己的情感，所以会言行不一，口是心非」

**解释是一切的核心**：对每一个性格概念进行解释，不要只贴标签。

- ❌ 「他是冷静的」
- ✅ 「他的冷静体现在：情绪管理能力优秀，情绪比常人更加稳定，不容易生气动怒，而不是像 nerd 一样只会说专有词汇、分析式的学术语言」

**一个条目只说一件事**：模型注意力有限，午饭场景不需要触发角色幼年被打的经历。

**关键词要「准」**：给每个条目设置 3-5 个相关触发词，提高命中率。

**内容要「精炼」**：成因链不是写长篇小说。每个层级用 2-3 句话说清楚因果。根据主题取舍——生存主题不需要写角色喜欢什么颜色的花。

**保留人味**：用户的原始想法和表达是最珍贵的素材——绝不替换用户的原文。帮助用户整理意图时，必须用用户自己的话重新写，不能把 AI 整合的内容直接拿来用。

**正面指导**：直接说明角色**应该做什么**，不要写「禁止角色 XXXX」。

# 角色补充设定

玩的时候觉得角色表现不对 → 单开一个条目（扮演核心 / 角色指导），用自然语言写。

行为表现和语料放在**对应性格下面**，不要单独分开放——语料和行为不是孤立的。

---

# 开场白

开场白是一张卡最重要的部分。

1. 介绍当前情况
2. 留钩子让玩家知道接下来怎么走剧情
3. 有期待感，知道大概玩什么、怎么走

注意规避 AI 爱用的重复句式和词语。用户自己的表达越多，模型能表现出来的就越好。
