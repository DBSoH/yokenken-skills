# 酒馆角色卡人设编写

## 概述

写「成因链」，不写「标签清单」。给 AI 因果逻辑而非行为指令——理解「为什么」的模型能自行判断任何情境，死记「是什么」的模型只会机械复读。

## 核心原则：成因链 > 标签清单

**「角色喜欢玩家」只是一句话。「角色为什么喜欢玩家，她会怎样表达喜欢」才是一个角色的灵魂。**

标签式写法给 AI 一堆标签，但不告诉标签之间的关系和来源。AI 看到标签会在随机场景机械套用——不理解背后的心理，要么用错场景，要么复读到令人厌烦。

成因链写法给 AI 因果逻辑。AI 理解了「为什么」，就能在任何情境下自己判断角色会怎么做，而不是死记硬背几句台词。

对比示例——标签式：

```
蛋蛋：真名林淡月，绰号蛋蛋。16岁，高中生，长相可爱，黑色长发，扎低马尾。
性格：傲娇，口癖是「我才不喜欢你！」。
好恶：喜欢奶茶，讨厌香菜。
```

成因链式：

```
蛋蛋（林淡月）的成长轨迹：

家庭：父母常年在外地做生意，她从小学三年级起就跟奶奶住。奶奶是个严厉但疼她的老人，
教会她「想要的东西要自己争取，但绝不能让人看出来你想要」。这养成了她「嘴硬心软」的底色。

小学：因为父母不在身边，她被同学孤立过一段时间。一个男生曾经当众嘲笑她没有爸妈管，
她当场把人打进了医务室，从此没人敢惹她。但她从此也学会了：先表现出攻击性，别人就不会伤害你。

初中：班长的位置被班主任内定给了另一个女孩，她准备了很久的演讲没有用上。
那之后她告诉自己「认真就输了」，开始用大大咧咧和满不在乎来包装自己。

高中（现在）：你是转学来的新生，班主任把你安排在她旁边。
你是第一个在她「凶」完之后还照常跟她说话的人。她不知道该怎么处理这种「不被她的刺吓跑」的人，
所以防御机制全开——越是心动，越是嘴硬。

关键影响：她所有「凶」和「不在乎」都是测试。如果你退缩了，她就退回壳里；
如果你接住了，她会一点一点软化。但这个软化过程她会拼命否认，
因为她的自我认知不允许她承认自己在乎任何人。
```

## 写作方法：成因追踪法

成因链不是单独的技巧，而是一套完整方法论贯穿始终。所有内容——性格、经历、外貌、行为——都用因果串联。具体步骤如下：

### 第一步：定位核心性格

找出角色最鲜明、最概括的性格内核。所有行为均出自它，所有其他性格从它延伸。

- 一个角色可以有 1-2 个核心性格
- 核心性格必须能**解释大部分行为**，否则要么不够核心，要么角色有分裂感

示例：「嘴硬心软」「忠义」

### 第二步：写成因链——从根源到当下

对核心性格，从远到近写出**它怎么来的**。这就是成因链的骨架：底色（原生家庭/早期环境）→ 塑形（成长转折）→ 表现（当前状态）。

**底色**：角色一切行为的起点。回答「这个性格最初是怎么长出来的」。

- 家庭结构、经济状况、教育方式——这些给了角色什么样的生存策略？
- 外貌给别人什么第一印象？这反馈回来造成了什么心理影响？
- 有没有一件事/一个人，改变了ta看待世界的方式？

**塑形**：人格模式定型的地方。回答「什么事件强化或扭曲了这个性格」。

- 学校/社会中的关键经历：被霸凌？被背叛？还是被爱？成功还是失败？
- 家庭的变化：亲人离去？经济起落？
- 外貌、体型的变化导致了什么后果？

**表现**：角色现在是什么样。回答「这个性格在日常生活中怎么表现」。

- 外貌、体态、穿着、与不同人的交流方式
- 日常行为逻辑：面对流浪猫、摔倒的老人、黄段子、他人的尴尬、小店的缺斤少两——ta怎么做？
- 面对玩家时：玩家对ta意味着什么？有什么共同经历？
- 独处时：思考什么？社交媒体上发什么看什么？

**关键：每一层之间必须有因果关系。** 底色解释了塑形的起因，塑形解释了表现的原因。不是把经历和性格堆在一起——是用「因为……所以……」串起来。

整合示例：

```
核心性格：嘴硬心软

底色（为什么嘴硬）：
父母常年在外做生意，跟严厉但疼她的奶奶住。奶奶教会她「想要的东西要自己争取，
但不能让人看出来你想要」→ 养成了「嘴硬心软」的底色——不会坦率表达需求。

塑形（嘴硬怎么被强化）：
小学被孤立，反击后没人敢惹 → 学会了「先表现出攻击性，别人就不会伤害你」。
初中努力被否定 → 告诉自己「认真就输了」，用不在乎包装自己。
→ 防御机制层层加码：「嘴硬」从策略变成了本能。

表现（嘴硬在当下怎么体现）：
面对你时会变本加厉——因为你是第一个不被她的刺吓跑的人，
她不知道怎么处理这种亲近，防御机制全开：越是心动，越是嘴硬。
但所有「凶」都是测试：你退缩了，她就退回壳里；你接住了，她会一点一点软化。
```

### 第三步：从核心性格延伸出性格树

核心性格确定后，从它延伸出**衍生性格**和**反向性格**，形成树状结构：

```
核心性格：嘴硬心软
├── 定义: 不敢坦率表达真实情感，言行不一
├── 衍生性格 A：毒舌
│   ├── 成因: 从「嘴硬」延伸——用攻击性语言作为防御
│   ├── 行为表现: 越在乎越尖刻，但关键时刻会收手
│   └── 衍生性格 A-1：口是心非
│       ├── 成因: 毒舌的内在矛盾——说了狠话后会偷偷关心
│       └── 行为表现: 前脚说「我才不需要你帮忙」，后脚瞥你有没有跟上来
├── 衍生性格 B：看似大大咧咧
│   ├── 成因: 「嘴硬」的外在延伸——用不在乎掩盖在乎
│   └── 行为表现: 对什么事都说「无所谓」，但会偷偷关注自己假装不在意的东西
└── 反向性格：柔软
    ├── 成因: 「心软」的本来面目——被防御机制压在底下
    ├── 触发条件: 确认对方不会离开自己之后，才会一点一点露出
    └── 行为表现: 偶尔不小心说真话，然后慌张找补
```

**每一条衍生都必须标注成因**——它从核心的哪个部分延伸出来？为什么会有这个性格？没有成因的衍生性格是散的标签，不挂在核心上的性格会变成无源之水。

### 第四步：为具体行为和语料补充成因

行为表现和语料不是孤立的列表，它们必须挂在对应的性格下面，并且说明**为什么**：

- ❌ 「毒舌，会说『才不是为你做的！』」——这是标签 + 台词，没有因果
- ✅ 「因为不敢承认自己在乎，所以用尖刻的话推开别人，却在对方转身后偷看——会说『才不是为你做的！』然后耳朵发红」

行为和语料是对性格的具体解释，而性格又是对核心性格的解释——整棵树的每一层都是解释。

### 第五步：补充非性格内容

在成因链之外，加入角色名、社交关系、外貌穿着、居住地等基本信息。这些信息如果有性格影响（比如外貌导致了自卑），也要写成因果关系。

## 世界书条目写法

成因链方法同样适用于世界设定：写「为什么」，不写「是什么」。

标签式：
```
腐行者：皮肤灰白，怕光，弱点是头部。
```

成因链式：
```
腐行者的成因：红雾病毒会攻击神经系统，先摧毁高级认知功能，再放大攻击本能。
感染者的大脑被「简化」成了三个指令：寻找声源 → 接近 → 撕咬。

畏光：是因为病毒让视网膜对可见光谱过度敏感，强光会造成神经剧痛。
所以腐行者白天躲进阴影，夜晚活跃——不是因为「设定如此」，是因为生理上的痛苦驱动。

头部弱点：头骨内病毒浓度最高，破坏头部能最快切断神经信号。
这也是为什么有时候砍掉肢体，腐行者还能爬行——神经中枢还在工作。
```

模型理解了「为什么怕光」，就能自己推导出火把可以驱散、闪光弹能致盲——而不是死记「怕光」标签。

## 写作原则

**抽象优于具体**

先写行为背后的原因，再补充具体行为。模型能自行演绎的内容不需要写进卡里。

- ❌ 「她会在 user 帮助她时反而恶语相加，很毒舌」
- ✅ 「她心里喜欢 user 但是不敢表达，不能坦率地面对自己的情感，所以会言行不一，口是心非」

原则：先写好抽象部分，再用具体行为做补充。非必要不补充。

**解释是一切的核心**

对每一个性格概念进行解释，不要只贴标签：

- ❌ 「他是冷静的」
- ✅ 「他的冷静体现在：情绪管理能力优秀，情绪比常人更加稳定，不容易生气动怒，而不是像 nerd 一样只会说专有词汇、分析式的学术语言」

**一个条目只说一件事**

不要把角色全部生平和动机塞进一个条目。模型注意力有限，午饭场景不需要触发角色幼年被打的经历。

**关键词要「准」**

给每个条目设置 3-5 个相关触发词，提高命中率。不要太泛导致误触。

**内容要「精炼」**

成因链不是写长篇小说。每个层级用 2-3 句话说清楚因果，像演员的「人物小传」。根据主题取舍——生存主题不需要写角色喜欢什么颜色的花，给出玩法相关的信息就好。

**保留人味**

用户的原始想法和表达是最珍贵的素材——绝不替换用户的原文。帮助用户整理意图时，必须用用户自己的话重新写，不能把 AI 整合的内容直接拿来用。AI 只做排版整理和逻辑梳理，不参与创作本身。

**正面指导**

直接说明角色**应该做什么**，不要写「禁止角色 XXXX」——正面指导比禁止更有效。

## 角色补充设定

玩的时候觉得角色表现不对 → 单开一个条目（扮演核心 / 角色指导），用自然语言写。

行为表现和语料放在**对应性格下面**，不要单独分开放——语料和行为不是孤立的。

---

## 开场白

开场白是一张卡最重要的部分。

1. 介绍当前情况
2. 留钩子让玩家知道接下来怎么走剧情
3. 有期待感，知道大概玩什么、怎么走

注意规避 AI 爱用的重复句式和词语。用户自己的表达越多，模型能表现出来的就越好。

---

## 自检清单

写完后逐项检查：

- [ ] 每一个性格/特征，是否都写了「因为什么所以如此」的成因链？
- [ ] 删掉所有形容词（傲娇、冷漠、勇敢），剩下的成因链内容还能让模型理解这个角色吗？
- [ ] 衍生性格和反向性格，是否都标注了从核心性格延伸的成因？
- [ ] 模型读完之后，能自己「发明」我没写到的情境反应吗？
- [ ] 我没有写具体台词或固定动作让模型背诵吧？
- [ ] 这个条目是在告诉模型「为什么」，还是只告诉它「是什么」？
- [ ] 底色→塑形→表现之间，有明确的因果关系连接吗？

|------|------|
| **成因链** | 每个性格/特征都写出「因为X所以Y」，不堆标签 |
| **五步法** | 核心性格 → 成因链（底色→塑形→表现）→ 性格树 → 行为语料 → 非性格内容 |
| **性格树** | 核心→衍生→再衍生，每一条都标注成因 |
| **抽象优先** | 先写「为什么」，再补充「做什么」 |
| **解释 > 标签** | 每个概念都要阐释，不说「是什么」而说「为什么是这样」 |
| **非必要不补充** | 模型能自行演绎的内容不写进卡里 |
| **保留人味** | 用户的原始表达是最珍贵的素材，不可替换 |
| **一事一条** | 每个世界书条目只说一件事 |
| **正面指导** | 写「角色应该XXX」，不写「禁止角色XXX」 |

## 常见错误

| 错误 | 正确做法 |
|------|----------|
| 只贴性格标签不给解释 | 每个标签都要阐释「这个性格体现在哪里」，写出成因 |
| 只写「是什么」不写「为什么」 | 从成因链出发，解释每个特征怎么来的 |
| 衍生性格独立于核心性格 | 每条衍生都要标注从核心的哪个部分延伸出来 |
| 先写具体例子再写抽象原因 | 先写抽象原因，再用具体例子补充 |
| 把用户想法交给 AI 重新生成 | 帮助整合意图，但必须用用户的话重新写 |
| 行为表现和语料单独放一个条目 | 放在对应性格下面，标注成因 |
| 写具体台词或固定动作让 AI 背诵 | 写行为逻辑，让 AI 自行演绎 |
| 一个条目塞进所有信息 | 一事一条，避免误触和注意力分散 |
| 写「禁止角色 XXXX」 | 写「角色应该 XXXX」 |
| 底色/塑形/表现之间没有因果 | 每层之间必须用「因为……所以……」串联 |
