# 世界书条目写法

成因链方法同样适用于世界设定：写「为什么」，不写「是什么」。

---

## 核心原则

模型理解了「为什么」，就能自己推导出新情境——而不是死记标签。

标签式：

```
腐行者：皮肤灰白，怕光，弱点是头部。
```

成因链式：

```
腐行者的成因：红雾病毒会攻击神经系统，先摧毁高级认知功能，再放大攻击本能。
感染者的大脑被「简化」成了三个指令：寻找声源 → 接近 → 撕咬。

畏光：是因为病毒让视网膜对可见光谱过度敏感，强光会造成神经剧痛。
所以腐行者白天躲进阴影，夜晚活跃——不是因为「设定如此」，是因为生理上的痛苦驱动。

头部弱点：头骨内病毒浓度最高，破坏头部能最快切断神经信号。
这也是为什么有时候砍掉肢体，腐行者还能爬行——神经中枢还在工作。
```

模型理解了「为什么怕光」，就能自己推导出火把可以驱散、闪光弹能致盲——而不是死记「怕光」标签。

---

## 写法要点

### 一个条目只说一件事

模型注意力有限，午饭场景不需要触发角色幼年被打的经历。把不同主题拆到不同条目里，各自设置精准的触发词。

示例——错误的条目：

```
关键词：蛋蛋、林淡月、高中、奶茶、嘴硬
内容：蛋蛋是林淡月的绰号，她今年16岁在XX高中上学，平时喜欢喝奶茶，
嘴硬心软是从奶奶那学来的……
```

这些应该拆成至少三个条目：角色基本身份、日常偏好、性格成因。

### 关键词要「准」

给每个条目设置 3-5 个相关触发词，提高命中率。不要太泛导致误触。

好的关键词：针对条目内容精准匹配
坏的关键词：把所有可能相关的词全塞进去

### 内容要「精炼」

成因链不是写长篇小说。每个层级用 2-3 句话说清楚因果，像演员的「人物小传」。根据主题取舍——生存主题不需要写角色喜欢什么颜色的花，给出玩法相关的信息就好。

### 写因果关系而非设定声明

每个世界书条目都应该回答「为什么这样」，而不是仅仅声明「是这样」。

| 写法 | 示例 |
|---|---|
| 设定声明 | 「XX 帝国的军队很强大」 |
| 成因链 | 「XX 帝国常年征战，军事贵族垄断朝政，从平民中征兵的制度让每个成年男性都有三年服役经验，所以军队的基层战斗力远超周边国家」 |
