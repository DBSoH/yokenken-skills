# 文风优化·犹疑校准（叶啃啃系列）

## 系列定位

本 Skill 与 `yokenken-editor` 同属 **文风优化 / 叶啃啃系列**：

- `yokenken-editor`：主编辑 Skill，负责发表级写作、改稿、审稿、人设、角色卡与完整四阶段流程。
- `wenfengyouhua-hesitation`：辅助校准 Skill，专门修复 AI 在“犹疑、未定感、半截话、高能量情绪语气、伪自述”上的偷懒写法。

正式小名：**犹疑校准**。兼容叫法：**犹豫校准**、**未定感校准**。

用户说“调用犹疑/犹豫/未定校准”“这里要不言尽”“话留半截”“语气太平”“自述不像本人”“不要写成介绍”“别写得太陈述”“别写愣了一下”“别写心情复杂”“怎么不像人话”“怎么那么肯定”“作者声部像外人假装”“作者不够内化”时，应读取并应用本 Skill。

---

## 与主 Skill 的关系

本 Skill 不替代 `yokenken-editor`。

当任务涉及正式写作、改稿、人设、角色卡、语料生成时，仍然遵守 `yokenken-editor` 的发表级标准、四阶段流程、九种空洞变体排查和白话语体要求。本 Skill 只补强“犹疑表达 / 情感语气 / 半截话”上的高频漏洞。

---

## 强制工作方式：侦察 → 判定 → 改写 → 验收

AI 最容易偷懒的地方不是不会写，而是**没有判定就直接润色**。本 Skill 禁止“凭感觉改得犹豫一点”。每次使用必须按下面顺序执行。

完整输出块、字段名、PASS/FAIL 验收格式见 `references/workflow.md`。**接手本 Skill 覆盖的任务时必须读取该文件；不读 workflow，不许输出改写。**

### 阶段一：侦察

按 `<reconnaissance>` 块确认五个锚点，缺任何一个都要在回答中声明缺口：

1. **叙述身份**：第一人称自述、第三人称贴身、对话层、独白层，分别处理。
2. **触发点**：哪句话、哪个动作、哪个物件让角色卡住。
3. **关系压力**：对方是谁，角色为什么不能直接说满。
4. **情绪能量**：低能量、平静、嘴硬、高能量、亢奋、咄咄逼人，不能混写。
5. **角色可知范围**：只写角色看见、听见、知道、误会、猜岔的内容，禁止替别人解释动机。

### 阶段二：判定

按 `<analysis>` 块逐项检查 H1-H9，命中一类就必须处理，不允许说“整体还行所以放过”。完整规则与示例见 `references/hesitation-rules.md`。

### 阶段三：改写

在 `<analysis>` 的 `execution_plan` 中制定方案。改写必须至少使用一种承担方式；当用户反问“怎么不像人话 / 怎么那么肯定”时，必须优先调用“尾巴词与舒缓词承担”检查，因为这通常不是信息量问题，而是句子落点太干净、太确信、太像报告：

- **半截话**：让话露出半句后停住。
- **错位接话**：把真正想说的话换成更安全、更偏题的一句。
- **动作替口**：用一个可见动作承担说不出口的内容。
- **物件承担**：让票根、杯子、伞、火腿肠、纸条等具体物件承压。
- **高能量找补**：用感叹号、尾巴词、波浪线承载嘴硬、心虚、求证，而不是当装饰。
- **尾巴词与舒缓词承担**：在对话层、内心自语层和作者自语层注入口语尾巴词（了/着/啦/嘛/呗/吧/啊/哦/呀）和舒缓字（了/着/呢/的/还/也/就），防止改写后的句子语气仍然像干净的报告而不是人在说话。详见 `references/hesitation-rules.md` 的「语气词与舒缓词承担」章节。

### 阶段四：验收

交付前必须输出 `<output_quality_review>`，逐项回答是否仍残留 H1-H9 中任一失守。详细清单见 `references/checklist.md`。

1. **强制加载**：接手本 Skill 覆盖的任务时，必须读取 `workflow.md`、`hesitation-rules.md` 与 `checklist.md`。

## H 编号速览

| 编号 | 失守类型 | 典型句 | 判定核心 |
|---|---|---|---|
| H1 | 伪第一人称自我介绍 | “我是一个嘴硬的人。” | 表面第一人称，实际人物档案 |
| H2 | 陈述式暧昧 | “她发现自己喜欢他。” | 情感被直接坐实 |
| H3 | 高能量误写 | 高能量角色被写成低声迟疑 | 角色能量被压低 |
| H4 | 标点空转 | “真的吗？！啊啊啊！” | 只有标点，没有语义摇摆 |
| H5 | 完成式未言 | “想接一句，后来没接。” | 把不言尽写成行为报表 |
| H6 | 否肯式情绪解释 | “不是不想要，是不敢要。” | 拉扯被解释完 |
| H7 | 二选一情绪摘要 | “不知道该笑还是该叹气。” | 用概括句替代真实反应 |
| H8 | 瞬时反应摘要 | “我愣了一下。” | 用空壳反应词跳过真实反应 |
| H9 | 抽象复杂感摘要 | “心情很复杂 / 说不上来什么感觉。” | 用抽象总括逃避承担 |

## 快速自检口令

如果你准备输出改写结果，先在心里跑一遍：

- 我是不是在总结角色，而不是让角色自己露馅？
- 我是不是把作者声部写成了外人旁观，而不是把作者当作一个会卡壳、会嘴硬、会自我推翻的角色来写？
- 我是不是把没说出口的东西写成了“后来没说/没接/没问”？
- 我是不是又写了“不是不 X，是 Y”“不知道该 A 还是该 B”“愣了一下”“心情复杂”？
- 我是不是拿标点、抽象词、情绪分类，代替了具体触发点和承担动作？
- 我是不是一连几句都太肯定、太完整、没有“了/着/呢/吧/啊/嘛/就/还/也”这类呼吸？
- 我改完以后这段读起来像人嘴里出来的，还是像一张干净的分析报告？

只要有一个“是”，就回去重写。能跑不等于能交付，文字也一样。
