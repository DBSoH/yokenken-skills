# 文风优化·犹疑校准验收清单

> **加载时机**：输出前必读。这个文件的作用不是提醒你“再看看”，而是阻止你把偷懒句子带出门。

---

## 强制输出块检查

除非用户明确要求“只给结果”，否则最终回答必须包含以下结构块：

1. `<reconnaissance>`
2. `<analysis>`
3. `## 改写结果`
4. `<output_quality_review>`

用户要求“只给结果”时，仍必须包含：

1. `## 改写结果`
2. `## 简短自检`

改写后检查：对话层、内心自语层和作者自语层是否携带了足够的尾巴词和舒缓词？如果语气偏报告腔，即使 H1-H9 都过了也要修。

### 必填字段检查

`<reconnaissance>` 必须包含：

- `goal`
- `input_scope`
- `narrative_layer`
- `trigger_point`
- `relationship_pressure`
- `energy_level`
- `knowledge_boundary`
- `missing_context`

`<analysis>` 必须包含：

- `context`
- `needs`
- `key_challenges`
- `confidence`
- `hit_summary`
- `findings`
- `non_hits_checked`，且覆盖 H1-H9。
- `affected_scope`
- `execution_plan`
- `degradation_check`

`<output_quality_review>` 必须包含：

- `residual_h_issues`，且覆盖 H1-H9。
- `forbidden_phrase_scan`
- `anchor_check`
- `format_check`
- `final_decision`

## 强制验收问题

交付前逐项回答，任何一项答案不是明确通过，都不能直接输出：

1. 是否把自述写成了人物档案？
2. 是否直接确认“喜欢/在意/期待/害羞/不敢/不能”？
3. 是否替对方解释动机？
4. 是否把心理活动切成一堆短段？
5. 是否用了“后来没接/没说/没问/没再讲”？
6. 是否用了“不是不 X，是 Y”？
7. 是否用了“不知道该 A 还是该 B”？
8. 是否用了“愣了一下 / 怔了一瞬 / 顿了顿 / 一时没反应过来”？
9. 是否用了“心情很复杂 / 说不上来什么感觉 / 很难形容”？
10. 是否只有标点，没有语义摇摆？
11. 是否保留角色原本的能量强度？
12. 是否有具体触发点和承担动作 / 物件 / 半截话？
13. 是否用了“我知道自己不该这样想 / 我明白这很可笑 / 我也说不好自己到底在期待什么”？
14. 是否用了“我平时不是……但这次……”式例外型自述（无具体触发点）？
15. 是否用了“也不知道从什么时候开始 / 说不上从哪一天起”式时间回避腔？
16. 对话层/内心自语层/作者自语层是否缺少尾巴词（了/着/啦/嘛/呗/吧/啊/哦/呀）？语气是否有报告腔？
17. 用户若反馈“怎么不像人话 / 怎么那么肯定”，是否已调用“语气词与舒缓词承担”完成硬落点、舒缓字密度、尾巴词扫描？
18. 作者声部是否被当成一个有限角色来写：有当场卡壳、嘴硬、自我推翻、舒缓词和尾音，而不是外人评价文本？
19. 作者声部是否出现表演性自我反思（“我意识到 / 我发现 / 我必须承认 / 我无法真正理解 / 沉默比语言 / 复杂性在于”）且没有具体语料、写作动作和自我推翻？

## 禁用摘要句黑名单

出现以下句子时，默认先判违规，再考虑极少数例外：

- “我当时不知道该笑还是该叹气。”
- “她不知道该高兴还是该难过。”
- “我愣了一下。”
- “她怔了一瞬。”
- “他说不上来是什么感觉。”
- “我心情很复杂。”
- “不是不想要，是不敢要。”
- “不是不喜欢，是不能喜欢。”
- “我想接一句，后来没接。”
- “我想说点什么，但最后什么都没说。”
- “我知道自己不该这样想。”
- “我平时不是会在意这种事的人，但……”
- “也不知道从什么时候开始，我开始……”
- “这段表达说明角色……”
- “这个角色的复杂性在于……”
- “作者需要保持克制地呈现……”
- “我意识到这个角色真正复杂的地方在于……”
- “我必须承认，我无法真正理解……”
- “我发现沉默比语言更能……”
- “沉默比语言更能说明……”

如果你真的想保留其中某类表达，必须证明它承担了具体触发点和局面，而不是摘要偷懒。

## 最小交付模板

当用户要求修一段文本时，按这个顺序输出：

1. `<reconnaissance>`：按 workflow 字段侦察上下文。
2. `<analysis>`：指出命中的 H 编号、原文证据、影响范围、执行方案和降级检查。
3. `改写结果`：给出完整文本，不省略。
4. `<output_quality_review>`：逐条确认 H1-H9 是否仍有残留。

---

## 快速口令

如果你已经写完一段，最后再问自己一次：

- 这像角色自己卡住了，还是像作者在替她总结？
- 这句话是反应，还是摘要？
- 我是不是又把过程省掉，直接给答案了？

只要有一点像后者，就回去重写。
