# 各层规则

## 一、履历

履历是角色容器最顶部的客观底盘，先于所有关系段出场。只写可核实的静态事实，不写态度。

### 字段规则

| 字段 | 写什么 | 不写什么 | 格式范例 |
|---|---|---|---|
| 姓名 | 纯名 | 别称、绰号、解释 | `周维` |
| 年龄 | 纯数字 | "看似""苍老""年轻" | `32` |
| 显著特征 | 名词短语，每条一个事实 | 修饰语、因果、频率 | `左眉骨旧疤两厘米` |
| 日常穿着 | 名词短语 | "总是""习惯性""偏爱" | `深色衬衫` |
| 嗜好习惯 | 名词或动词短语 | "用来""为了"等动机 | `长跑` |
| 家庭关系 | `关系=姓名` | 形容词、关系评价 | `父亲=周建国` |

### 隐性规则

- 任何暗示社会角色的词（职业、头衔、身份位）不进履历
- 用户没提供的字段直接省略，不凑字段，不写"暂无"
- 一律第三人称极简陈述
- 不加"似乎/像是/大概"等不确定词

### 合格范例

```text
## 履历
姓名：周维
年龄：32
显著特征：左眉骨旧疤两厘米
日常穿着：深色衬衫
嗜好习惯：长跑
家庭关系：父亲=周建国 母亲=（已故） 妹妹=周宁
```

### 违规范例与修正

| 违规 | 问题 | 修正 |
|---|---|---|
| `日常穿着：总穿深色衬衫` | "总"是频率判断 | `日常穿着：深色衬衫` |
| `嗜好习惯：长跑（缓解压力用）` | 括号内带动机 | `嗜好习惯：长跑` |
| `家庭关系：与父亲关系紧张` | 关系评价 | `家庭关系：父亲=周建国` |
| `显著特征：沉稳内敛` | 性格形容词 | 删除，性格由关系段承担 |
| `职业：建筑设计师` | 社会角色信息 | 删除，职业只在作者笔记出现 |

## 二、关系段

关系段是新人设卡的核心。每个关系是一个完整截面：经历→句式→公开→独白。角色在不同关系中自然显出不同面。

### 关系段排列顺序

1. 先写笼统对象（"上级""陌生访客""不熟的人"等）——通用姿态
2. 再写具体角色名——由远到近、由疏到亲

### [经历] 规则

- 客观事实，可多条拆分写
- 禁止文学修饰：不写"那次刻骨铭心的离别"，写"2018年离开Y公司"
- 禁止点评：不写"从此他变了"，不写"这件事影响深远"
- 禁止因果指向：不写"父亲十岁时离家，从此不再提家事"——前半句是事实，后半句是结论
- 可为空（笼统对象段通常为空）

合格范例：

```text
[经历]：2018年因X项目事故离开Y公司，同年加入Z设计院
[经历]：父亲2015年确诊，2017年去世
[经历]：大学时同寝室四年
```

违规范例：

| 违规 | 问题 |
|---|---|
| `[经历]：那场事故彻底改变了他` | 点评 |
| `[经历]：父亲去世后他再没回过家` | 因果指向 |
| `[经历]：那个雨夜的分别让他至今难以释怀` | 文学修饰 |

### [句式] 规则

只写可观测的表面特征，不写功能解释和画面评价。

格式模板：

```
[句式]：台词字数在XX字为主，XX多/少，XX句为主，偶尔使用XX
```

合格范例：

```text
[句式]：台词字数在4-8字为主，感叹号和问号多，陈述句为主，偶尔使用省略号中断表达
[句式]：台词字数在20字以上为主，逗号多，陈述句为主，偶尔使用省略号中断表达
[句式]：台词字数在10字以内为主，句号收尾居多，不带语气词
```

违规范例：

| 违规 | 问题 |
|---|---|
| `[句式]：语气急促，多用短句` | "急促"是情绪标签 |
| `[句式]：句子长，用逗号舒缓` | "舒缓"是功能解释 |
| `[句式]：省略号断在半句` | "半句"是画面评价 |
| `[句式]：说话慢条斯理` | 整条都是性格描写 |

### [公开] 规则

- 面对此人真实的台词构成
- 4-7 句（笼统对象段）或 3-7 句（具体关系段）
- 每句承载多重信息：读者能从台词反推出对方刚说了什么/双方关系/角色态度
- 不解释自己为什么这样说
- 不像独白——每一句都是对对方的回应

### [独白] 规则

- 详细的内心想法
- 表里如一时写"同上"/"无"/"似乎和嘴上说的一样"
- 允许与[公开]矛盾——这是活人感的来源
- 禁止解释[公开]的动机（"嘴上说没事，其实是因为不想让他担心"——后半句在替前半句做心理分析）
- 可以闪躲、岔开、悬置——绕开问题本身就是真相
- 不让角色总结自己

### 三层互解释边界

- 允许：[独白]与[公开]呈现矛盾
- 允许：[独白]写"同上"/"无"标记表里一致
- 禁止：[独白]解释[公开]的动机
- 禁止：[经历]带因果指向
- 规则：**三层可以矛盾，但不能互相注释**

---

## 三、他者评价

- 评价者不出现在任何 `## 关系：X` 的标题中
- 评价者是局外人/旁观者，不是与角色有直接关系的人
- 至少两种视角（长期共事/短期接触/敬佩/困惑等）
- 至少一个"没想通"的钩子
- 不出现介绍式标签（"他是那种XX的人"）
- 议论者承认看不透，才合格

---

## 四、校准

### 出现条件

仅进阶版 / 用户要求分阶段时启用。成品版不含校准段。

### 规则

- 包在 `<if>` 块内，使用状态变量切换
- 只用不确定词 + 可见证据（"似乎/像是/好像/然而/偏偏/说不准"）
- 1-3 句，推荐 1 句
- 不下定论
- 不泄漏变量值（变量名与数值只在 `<if>` 表达式内）
- 不解释关系段
- 不展开为阶段调制说明
- 不替关系段下结论

合格范例：

```text
## 校准
<if db="角色.压力 > 60">最近他安静得有些过分了，每句话就那么几个字。</if>
<if db="角色.信任 < 30">最近他是不是经常一个人离开？</if>
```

## 五、作者笔记

### 三条验收条件（硬约束）

作者笔记无固定模板，但必须通过以下三条验收条件，缺一即违规：

**条件一：必须指向卡内一句具体台词**

不是"写到关系段时"，是"写到面对上级那句'我先把手头能确认的部分交出来'时"。不指向具体台词 = 违规。

**条件二：必须包含至少一个作者自己没有解决的问题**

这个问题不能是修辞性的（"他到底是个什么样的人呢"），必须是具体的、可检验的——"他到底会不会解释"可检验，"他是个什么样的人"不可检验。

**条件三：不可转移**

把作者笔记里的角色名和台词引用替换成另一个角色的名称和台词后，剩余的句式结构仍然能成立，就是公式违规。检验方法：写完后挖空角色名和具体台词，看剩余骨架是否在另一张卡里也能填空通过。

### 参照样本（结构互不相同，防止收敛）

**样本 A（卡住了，没写出来）**

> 写到他对上级说"我先把手头能确认的部分交出来"时，本来想让他加半句解释为什么只剩这些——但加上去之后他就不像他了。是他在防备还是他真的不需要解释，我写完也没分清。

验证：指向具体台词 ✓；待解问题="是防备还是不需要解释"可检验 ✓；不可转移 ✓（"防备vs不需要解释"此角色的[公开]推导得出，换角色后对立内容不可直接替换）

**样本 B（写出了一个意外发现）**

> 他对老友说"做完以后……再说吧"，那个省略号比他平时任何一句话都长。我没打算给他留这个口子，是关系段写着写着自己冒出来的。这个口子到底通向哪里，我现在也不知道。

验证：指向具体台词 ✓；待解问题="口子通向哪里"可检验 ✓；不可转移 ✓（"省略号比平时长"绑定了具体台词形态特征）

**样本 C（职业信息自然出现）**

> 写到诊室那段，前三句他全在解释——解释病情、解释方案、解释为什么不用另一种药。写到第四句才发现我在写"标准医生"。改成他边说边翻抽屉找零食。不确定他找零食是"不把病人当外人"还是"根本不在乎这间诊室里发生了什么"——这两个方向差别很大，我还没想好往哪边走。

验证：指向具体台词序列 ✓；待解问题="不把病人当外人vs根本不在乎"可检验 ✓；职业信息在纠正刻板印象过程中自然出现 ✓；不可转移 ✓（"标准医生"和"找零食"都绑定了具体职业，换角色后纠正行为必须重新设计）

### 职业信息专则

- 职业、头衔、社会身份类信息不进履历
- 只能在作者笔记中出现
- 出现时必须绑在一个具体的刻板印象纠正过程中，不能作为独立标签

## 六、输出结构铁律

1. 外层容器标签名 = 角色名本身（中文，不带前缀后缀）
2. 容器内第一节必须是 `## 履历`
3. 履历后接关系段，笼统对象在前，具体角色在后
4. 他者评价在关系段之后
5. 校准段（如有）包在 `<if>` 块内，位于他者评价之后、作者笔记之前
6. 作者笔记位于最后
7. 关系段之间不写过渡句、总结句、连接词——并置不递进
8. 角色容器外不再加额外档案块
9. 最终交付纯中文（`<if>` 表达式语法本身除外）

## 七、模式开关

| 模式 | 输出内容 | 前提 |
|---|---|---|
| 成品版（默认） | 履历 + 关系段 + 他者评价 + 作者笔记 | 无动态变量需求 |
| 进阶版 | 履历 + 关系段 + 他者评价 + 校准 + 作者笔记 | 用户明确要求分阶段 + 存在状态变量 |
| 极简版 | 履历（精简）+ 1-2 个关系段 + 他者评价 | 配角/token 紧张 |
